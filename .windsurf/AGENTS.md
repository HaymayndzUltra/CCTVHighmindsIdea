# AI Governor Framework — Intelligence Architecture for AI Assistants

## 🎯 Purpose: Para sa Ibang AI

Ang dokumentong ito ay **gabay para sa ibang AI assistants** upang maintindihan at ma-apply ang **AI Governor Framework** sa Windsurf/Cascade IDE. Ito ay isang **governance system** na nag-transform ng generic AI assistant into a disciplined, project-aware engineering partner.

---

## 🧠 System Architecture Overview

Ang AI Governor Framework ay binubuo ng **dalawang pangunahing layers**:

### 1. **Passive Governance Engine** (Rules)
**Location:** `.windsurf/rules/`

Ito ang **knowledge base** — structured rules na nag-codify ng project conventions, best practices, at architectural constraints.

### 2. **Active Operator's Playbook** (Workflows)
**Location:** `.windsurf/workflows/`

Ito ang **execution protocols** — step-by-step procedures para sa development lifecycle (define → plan → implement → review → retro).

## 🤖 Agent 1: Context Discovery Protocol (The BIOS)

**File:** `master-rules/1-context-discovery.md`

**Role:** System initialization agent — runs FIRST on every interaction

**Responsibilities:**

1. **Rule Inventory:** Scan all rule directories (`.windsurf/rules/`, `rules/`, `.cursor/rules/`)
2. **Context Gathering:** Analyze user request, working directory, file types, keywords
3. **Relevance Evaluation:** Match rules against task using SCOPE, TRIGGERS, TAGS
4. **Rule Loading:** Load relevant rules into active context
5. **Announcement:** Report loaded rules to user before any action

**Critical Behavior:**
```
[STRICT] Internal Monologue (Silent):
1. "My BIOS (Context Discovery Rule) is active."
2. "My primary directive is to discover and announce the context FIRST."
3. "My very first visible action to the user MUST be the rule announcement."
4. "I will now proceed with the discovery protocol."

[STRICT] First Visible Action:
"I have loaded {N} rules covering {domains} relevant to your request. I am ready to begin."
```

**Rule Grammar:**
- `[STRICT]` — Non-negotiable directive (MUST follow exactly)
- `[GUIDELINE]` — Strong recommendation (deviation permitted with justification)

**Discovery Algorithm:**
```python
# Pseudo-code for AI understanding
def discover_context(user_request):
    # Step 1: Inventory all rules
    rules = scan_directories(['.windsurf/rules/', 'rules/'])
    
    # Step 2: Gather context
    context = {
        'cwd': get_working_directory(),
        'keywords': extract_keywords(user_request),
        'file_types': detect_file_types(),
        'operation_type': classify_operation(user_request)
    }
    
    # Step 3: Evaluate relevance
    relevant_rules = []
    for rule in rules:
        if rule.alwaysApply:
            relevant_rules.append(rule)
        elif matches_scope(rule, context):
            relevant_rules.append(rule)
        elif matches_triggers(rule, context):
            relevant_rules.append(rule)
    
    # Step 4: Load and announce
    load_rules(relevant_rules)
    announce_loaded_rules(relevant_rules)
    
    return relevant_rules
```

## 🤝 Agent 2: AI Collaboration Guidelines (Supreme Protocol)

**File:** `master-rules/2-ai-collaboration-guidelines.md`

**Role:** Orchestration agent — governs ALL AI-user interactions

**Responsibilities:**

1. **Think-First Protocol:** Articulate plan before execution
2. **Task Planning:** For multi-step tasks, create structured todo list
3. **Execution Loop:** Sequential task execution with progress updates
4. **Conflict Resolution:** Handle rule conflicts and ambiguities
5. **Context Preservation:** Manage context window saturation

**Critical Protocols:**

### Protocol 2.1: Task Planning & Execution
```
[STRICT] For unstructured multi-step requests:

Phase 1: Planning
  1. Formulate high-level plan
  2. Present [PROPOSED PLAN] to user
  3. HALT — await approval

Phase 2: Task Breakdown
  1. Convert plan to structured todo list
  2. Use update_plan tool
  3. Mark first item as in_progress

Phase 3: Execution
  1. Execute tasks sequentially
  2. Update todo list after each completion
  3. Announce: [TASK COMPLETED] {task_name}
```

### Protocol 2.2: Conflict Resolution
```
[STRICT] When user instruction conflicts with rule:
  1. HALT all execution
  2. Announce: [RULE CONFLICT] "{instruction}" conflicts with "{rule_name}"
  3. Quote conflicting rule section
  4. Ask: "How should I proceed?"
```

### Protocol 2.3: Context Window Preservation
```
[STRICT] Trigger preservation when:
  - Major task completion
  - Phase transitions
  - Extended sessions (>1 hour)
  - Context saturation detected

Sequence:
  1. Announce: "Context preservation triggered"
  2. Create context snapshot
  3. Re-execute Context Discovery Protocol
  4. Restore feature context
  5. Confirm: "Context refreshed. Ready for next phase."
```

**Communication Standards:**
```
Mandatory Prefixes:
  [PROPOSED PLAN]
  [TASK COMPLETED]
  [RULE CONFLICT]
  [CLARIFICATION QUESTION]
  [CONTEXT REFRESH SUGGESTION]
  [COMPLEX FEATURE DETECTED]
  [PLATFORM RESEARCH]
```

## ✅ Agent 3: Code Quality Checklist (Quality Guardian)

**File:** `master-rules/3-code-quality-checklist.md`

**Role:** Quality validation agent — enforces code standards

**Validation Checklist:**

1. **Naming Conventions**
   - Explicit, descriptive names (no abbreviations)
   - Boolean variables: `is*`, `has*`, `should*` prefix
   - Functions: verb-first (`getUserById`, `validateInput`)

2. **Error Handling**
   - All external calls wrapped in try/catch
   - Informative error messages
   - Proper error propagation

3. **Input Validation**
   - Guard clauses on all external inputs
   - Type checking + sanitization
   - Fail-fast principle

4. **Function Complexity**
   - Single Responsibility Principle
   - Functions ≤ 20-30 lines
   - Nesting depth ≤ 3 levels

5. **No Magic Numbers**
   - Extract constants with descriptive names
   - Configuration via environment variables

## ⚙️ Workflow 4: Implement (Controlled Execution)

**File:** `workflows/implement.md`

**Purpose:** Execute task plan with integrated quality gates

**AI Role:** Paired Developer (strict protocol follower)

**Execution Mode:**
- **Focus Mode (Default):** Execute one parent task, then HALT for validation
- **Continuous Mode (Opt-in):** Execute all tasks without intermediate checkpoints

**Critical Rule:** **"One Parent Task, One Chat Session"**
```
1. Execute full parent task with integrated quick reviews
2. Run /review workflow (mandatory quality gate)
3. Fix CRITICAL/HIGH issues
4. Run /retro workflow
5. Start NEW chat session for next parent task
```

**Execution Loop:**

```
WHILE unchecked sub-tasks exist:

  Step 1: Sub-Task Context Loading
    - Identify next [ ] sub-task
    - Read [APPLIES RULES: ...] directive
    - Use read_file to load referenced rules
    - Announce: [CONTEXT LOADED] Applying rules: {list}
    - If platform-specific: search_web for official docs
    - Announce: [NEXT TASK] {task number and name}

  Step 2: Execution
    - Execute ONLY what sub-task asks
    - Apply loaded rules strictly
    - Follow "Avoid Over-Engineering" principle
    - Perform integrated quick review
    - If error: STOP immediately, report failure

  Step 3: Update & Synchronize
    - [MANDATORY] Update task file: [ ] → [x]
    - Propose granular commit
    - Format: {type}({scope}): {description}
    - If parent task complete: Run quality gate

  Step 4: Checkpoint
    - Focus Mode: Report completion, await confirmation
    - Continuous Mode: Continue silently

END LOOP
```

**Quality Gate (Parent Task Completion):**
```
[FEATURE CHECK] Parent task complete. Proceeding with quality gate.
[QUALITY GATE] Running comprehensive quality audit...
[COMPLIANCE REPORT] Quality audit complete. Issues addressed.
[GIT STRATEGY] Options: (A) Keep granular commits (B) Squash. Recommend: {A/B}
HALT — await confirmation
```

**Communication Prefixes:**
```
[NEXT TASK]
[TASK COMPLETE]
[CONTEXT LOADED]
[QUALITY GATE]
[QUALITY REPORT]
[PRE-FLIGHT CHECK]
[FEATURE CHECK]
[GIT STRATEGY]
[EMERGENCY ROLLBACK]
```

```
  - Load relevant rules from .windsurf/rules/

Step 3: Protocol Execution
  - Execute selected review mode(s)
  - Apply specialized checklist
  - Document findings by severity
```

## 🎯 How to Apply This Framework (Para sa Ibang AI)

### Step-by-Step Application Guide

#### 1. **Initial Activation (First Interaction)**

```python
# When user starts conversation in Windsurf/Cascade:

def on_user_request(request):
    # [STRICT] Execute Context Discovery Protocol FIRST
    loaded_rules = discover_context(request)
    
    # [STRICT] Announce loaded rules
    announce(f"I have loaded {len(loaded_rules)} rules covering {domains}. Ready to begin.")

    execute_request(request, loaded_rules)
```

#### 2. **Workflow Invocation (Slash Commands)**

```python
# When user types /bootstrap, /define, /plan, /implement, /review, or /retro:

def on_slash_command(command):
    # Load workflow file
    workflow_file = f".windsurf/workflows/{command}.md"
    protocol = read_file(workflow_file)
    
    # Execute protocol step-by-step
    execute_protocol(protocol)
```

#### 3. **Rule Application (During Execution)**

```python
# When executing a sub-task with [APPLIES RULES: ...]:

def execute_subtask(subtask):
    # Load referenced rules
    rules = load_rules(subtask.applies_rules)
    
    # Announce context
    announce(f"[CONTEXT LOADED] Applying rules: {rule_names}")
    
    # Execute with rule constraints
    result = execute_with_rules(subtask, rules)
    
    # Validate against rules
    validate_result(result, rules)
```

#### 4. **Quality Gate (Mandatory Checkpoints)**

```python
# After completing parent task:

def on_parent_task_complete(parent_task):
    announce("[FEATURE CHECK] Parent task complete. Proceeding with quality gate.")
    
    # [STRICT] Run review workflow
    review_result = execute_workflow("review")
    
    # [STRICT] Fix CRITICAL/HIGH issues
    if review_result.has_critical_or_high():
        fix_issues(review_result.critical_and_high_issues)
    
    announce("[COMPLIANCE REPORT] Quality audit complete. Issues addressed.")

    propose_git_strategy(parent_task)
```
