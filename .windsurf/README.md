# AI Governor Framework — Windsurf Integration Guide

## Overview

This `.windsurf/` directory contains the complete **AI Governor Framework** adapted for the Windsurf/Cascade IDE. It provides a structured governance system that transforms your AI assistant into a disciplined, project-aware engineering partner.

## Quick Start

### Step 1: Bootstrap Your Project (One-Time)
```
/bootstrap
```
This analyzes your codebase, generates architectural documentation (`README.md` files), and creates project-specific governance rules tailored to your tech stack.

### Step 2: Development Lifecycle
For each new feature, follow the governed lifecycle:

| Step | Command | What Happens |
|:---|:---|:---|
| 1 | `/define` | AI interviews you to create a detailed PRD |
| 2 | `/plan` | AI transforms the PRD into a granular task list |
| 3 | `/implement` | AI executes tasks with integrated quality gates |
| 4 | `/review` | AI runs a multi-layer quality audit |
| 5 | `/retro` | AI extracts learnings and improves the governance system |

### Ad-Hoc Reviews
Run `/review` at any time to perform an on-demand quality audit with mode selection:
- **Code Review** — Quick code quality check
- **Security Check** — Auth, data validation, secrets
- **Architecture Review** — Design, performance, dependencies
- **Design System** — Token compliance, theming, responsive
- **UI/Accessibility** — WCAG AA, keyboard navigation, ARIA
- **Pre-Production** — Final security gate before deploy
- **Run All** — Comprehensive 6-layer validation

## How Rules Work

### 3-Layer Hierarchy
1. **Foundation (BIOS):** Rules 1-2 are `alwaysApply: true` — they run on every interaction, establishing context discovery and collaboration protocols.
2. **Guardians:** Rules 3-6 activate when relevant — code quality, modification safety, documentation integrity, and rule creation guidance.
3. **Experts:** Common rules activate conditionally when the task domain matches (UI design, accessibility, enterprise features).

### Rule Grammar
- `[STRICT]` — Non-negotiable directive. Must follow exactly.
- `[GUIDELINE]` — Strong recommendation. Deviation permitted with explicit justification.

### Creating New Rules
Use the guide in `rules/master-rules/6-how-to-create-effective-rules.md` or let `/bootstrap` generate them automatically.
